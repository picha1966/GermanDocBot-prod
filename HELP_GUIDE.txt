## ⚡ Fast Installation (3 steps)

### Step 1: Backup your database
```bash
cp bot_database.db bot_database.db.backup_$(date +%Y%m%d_%H%M%S)
```

### Step 2: Replace files

**A) Replace `database.py`:**
- Copy code from artifact `universal_analytics_db`
- Save as `backend/database.py` (or wherever your database.py is)

**B) Replace GDPR handler in `bot.py`:**
- Find function `handle_gdpr_callback` in your bot.py
- Replace it with code from artifact `gdpr_handler_guaranteed`

### Step 3: Restart bot
```bash
# Stop bot
pkill -f bot.py

# Delete old database to recreate structure (OPTIONAL - only if you want fresh DB)
# rm bot_database.db

# Start bot
python3 bot.py
```

## 📊 Database Schema

The new `analytics_events` table:
```sql
CREATE TABLE analytics_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    event_type TEXT,      -- Main event identifier
    event_name TEXT,      -- Alternative name (for compatibility)
    event_data TEXT,      -- JSON data
    created_at TEXT       -- ISO timestamp
);
```

## 🔧 Troubleshooting

### Problem: `TypeError: unexpected keyword argument`
**Solution:** You're using OLD database.py. Replace with new universal version.

### Problem: GDPR accepted but no menu appears
**Solution:** 
1. Check if `menu.py` has `show_main_menu()` function
2. Check imports in bot.py: `import menu`
3. Look at logs for errors

### Problem: `sqlite3.OperationalError: no such column`
**Solution:** Delete old database and restart:
```bash
rm bot_database.db
python3 bot.py
```

### Problem: Analytics not logging
**Check database:**
```bash
sqlite3 bot_database.db "SELECT * FROM analytics_events ORDER BY id DESC LIMIT 5;"
```

**Check if table exists:**
```bash
sqlite3 bot_database.db ".schema analytics_events"
```
